# 角角桌宠 · 功能规划

## 1. 专属哈气素材（yawning）

**现状**：`yawning` 状态复用 `sleeping.png`，视觉上看不出区别。

**目标**：生成专属哈气精灵图，嘴巴微张、眼睛眯起、耳朵微压。

**实现步骤**：

1. 用 gpt-image-2 生成精灵图，参考 prompt：

   > 像素风蓝灰色英国短毛猫，单行横向 8 帧，图像总尺寸 2048×768，每帧 256×768。猫咪正在打哈欠：第1帧嘴微张、眼睛开始眯；第2-4帧嘴巴张到最大、眼睛完全闭合；第5-8帧嘴慢慢合拢、眼睛重新睁开。白色纯色背景，猫体居中，不贴近帧边缘。

2. 保存为 `source_sprites/yawning.png`

3. 在 `make_apng.py` 的 `STATES` 里添加：
   ```python
   "yawning": {
       "file": "yawning.png",
       "cols": 8, "rows": 1,
       "n_frames": 8,
       "ms": 300, "pingpong": True,
   },
   ```

4. 在 `theme.json` 的 `states` 里把 `yawning` 从复用改为独立文件：
   ```json
   "yawning": ["yawning.png"]
   ```

5. 重新运行 `python make_apng.py`

## 2. 午饭状态（eating）

**现状**：没有时间感知，角角全天行为一致。

**目标**：11:45–13:00 之间随机触发一次低头吃饭动画。

**实现步骤**：

1. 生成 `eating` 精灵图，参考 prompt：

   > 像素风蓝灰色英国短毛猫，单行横向 8 帧，2048×768。猫咪低头吃东西：头慢慢低下→嘴巴触碰碗边→抬头咀嚼→再次低头，循环。白色背景，猫体居中。

2. `make_apng.py` 加入 `eating` 条目（同上）

3. `theme.json` 加入状态和触发时间窗口（需确认 clawd-on-desk 是否支持 `scheduleWindows` 字段，若不支持则在本地脚本层做轮询）：
   ```json
   "eating": ["eating.png"],
   ```
   ```json
   "scheduleAnimations": [
     {
       "state": "eating",
       "timeStart": "11:45",
       "timeEnd": "13:00",
       "oncePer": "day"
     }
   ]
   ```
   > ⚠️ 若 clawd-on-desk 不支持此字段，需要写一个独立脚本定时调用其 IPC/WebSocket 接口触发状态切换，或等上游支持后再实现。

## 3. 拖动挣扎动画（drag resistance）

**现状**：拖动角角时无特殊反应。

**目标**：被拖动时播放四肢乱蹬、表情抗拒的动画。

**实现步骤**：

1. 确认 clawd-on-desk 是否暴露 `onDragStart` / `onDragEnd` 钩子（查阅官方文档或源码）

2. 若支持，生成 `dragging` 精灵图：

   > 像素风蓝灰色英国短毛猫，单行横向 8 帧，2048×768。猫咪被抱起时挣扎：身体略微倾斜，四肢乱蹬，耳朵压平，眼神不满。白色背景。

3. `theme.json` 加入：
   ```json
   "dragging": ["dragging.png"]
   ```
   并在 reactions 或 hooks 字段绑定到拖拽事件（字段名待确认）

4. 若 clawd-on-desk 暂不支持，记录为 upstream feature request，先搁置。

## 4. 报错情绪积累（error persistence）

**现状**：每次报错独立触发 error 状态，结束后立刻恢复，无累积效果。

**目标**：连续报错 N 次后，error 状态持续时间延长，表现出明显的"烦躁积累"。

**实现步骤**：

1. 在 `theme.json` 的 `timings` 或新增 `errorPersistence` 字段配置阈值（待确认字段名）：
   ```json
   "errorPersistence": {
     "threshold": 3,
     "extendedDuration": 10000
   }
   ```

2. 若 clawd-on-desk 不支持原生配置，可在外部维护一个计数器脚本：
   - 监听 Claude 的错误输出（通过 clawd-on-desk 的日志或 hook）
   - 连续报错超过阈值后，通过 IPC 强制锁定 error 状态一段时间
   - 下一次成功响应后解锁

3. 可考虑加一个 `angry` 状态（比 error 更夸张的表情），专门用于积累后触发，以示区别。

   精灵图 prompt 参考：
   > 像素风蓝灰色英国短毛猫，单行横向 8 帧，2048×768。猫咪非常生气：耳朵完全压平，眉头紧皱，嘴角下撇，尾巴用力拍地，轻微发抖。白色背景。
